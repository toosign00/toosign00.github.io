#### "프론트엔드 개발자로서 사용자 경험을 중심으로 한 끊임없는 도전과 성장을 이어나가고 있습니다."

여러 전공 수업을 듣던 중, 웹퍼블리싱 수업을 통해 프론트엔드 개발이라는 직무를 알게 되었습니다. 이 과정을 통해 HTML, CSS, JavaScript를 사용하여 개인 포트폴리오 사이트를 제작하며, 사용자 경험(UX)의 중요성을 깨달았습니다. 웹사이트를 제작하는 과정에서 문제를 해결하는 경험을 통해 지금까지 느껴보지 못한 성취감을 얻게 되었습니다.

2학년 때 프로그래밍 세부 전공을 선택한 후, 프론트엔드 개발자가 되겠다는 목표를 세웠습니다.  
프론트엔드 개발은 단순히 코드 작성에 그치지 않고, 사용자와의 상호작용을 통해 더욱 직관적이고 유익한 경험을 제공하는 일이라고 생각합니다. 목표를 이루기 위해 기초부터 차근차근 공부해 나가고 있습니다. 이 과정에서 프로그래밍 언어에 대한 궁금증이 커졌고, 단순히 언어를 배우는 데 그치지 않고 다양한 CS 지식도 습득하는 데 큰 흥미를 느꼈습니다. 예를 들어, 제가 사용하는 언어가 왜 생겨났는지, 웹사이트가 어떻게 작동하는지, 자바스크립트 엔진의 동작 원리 등을 공부하면서 단순히 코드를 작성하는 개발자가 아닌 목적과 원리를 이해하는 개발자로 성장하고자 합니다.

저는 남들보다 조금 느리더라도 천천히 토대를 다지고 점진적으로 성장해 나가는 프론트엔드 개발자가 되기 위해 노력하고 있습니다. 제 경험과 열정을 통해 여러분의 회사에 긍정적인 기여를 하고, 프론트엔드 개발자로서의 여정을 계속 이어가고 싶습니다.

#### "끈기와 집요함으로 사용자 경험을 최적화하는 프론트엔드 개발자입니다."

프로젝트를 진행하는 과정에서 여러 어려움이 있었습니다. 그중 하나는 웹사이트에 다크모드 테마를 제작하는 과정에서 발생했습니다. localStorage를 사용해 사용자가 선택한 테마 값을 저장하고, 그 값을 통해 다른 페이지로 이동하더라도 테마가 유지되도록 기능을 개발하던 중, 페이지가 전환될 때 라이트모드가 먼저 로드되고 다크모드가 나중에 로드되어 화면이 번쩍이는 현상이 발생하였습니다.

이 문제를 해결하기 위해 body가 렌더링되기 전에 localStorage 값에 따라 배경색을 미리 로드하는 스크립트를 작성하여 이 문제를 해결하였습니다. 이 과정에서 개발과 디자인을 병행하며 사용자의 경험을 최적화하는 것이 얼마나 중요한지 깨닫게 되었습니다.

문제 해결 과정은 2주 정도 걸렸습니다. 하지만 저는 중간에 포기하거나 타협하지 않고 끝까지 기능 개발을 완료했습니다. 이처럼 저는 문제에 대해 집요하게 파고들고, 끈기 있게 문제를 해결할 수 있다는 것을 보여주었습니다. 이 경험은 저의 프론트엔드 개발자로서의 자질을 더욱 강화시켰으며, 특히 사용자 중심의 문제 해결 능력을 길러주었습니다.

#### "저는 프론트엔드 개발자로서 팀원들과의 원활한 소통과 협업을 통해 프로젝트의 성공적인 완수를 이끌어내는 개발자입니다."

여러 프로젝트 중 팀 프로젝트를 진행할 때 팀장을 맡지는 않았지만, 팀의 일원으로서 단순히 기획이나 디자인 파트의 팀원들이 가져다주는 시안만 코딩하는 것이 아닌, 기획, 디자인 단계에서부터 개발의 관점에서 여러 의견과 아이디어를 제시하며 소통하기 위해 노력하였습니다. 이를 통해 팀원들과의 원활한 협업을 이끌어내어 프로젝트의 완성도를 높일 수 있었습니다.

특히 팀의 개발 담당 팀원과 협업을 위해 Git과 GitHub를 적극적으로 사용하여 코드 상태 관리를 했습니다. 브랜치 보호 규칙, 코드 작성 규칙, Pull Request 사용 등을 통해 개발 과정을 주도하며 팀원과의 협업이 원활하게 이루어지도록 노력했습니다. 이 과정에서 소통과 협력 능력을 기를 수 있었습니다.

팀 내에서의 원활한 소통과 협업은 프로젝트의 성공적인 완수에 큰 기여를 했습니다. 저는 항상 팀원들의 의견을 경청하고, 각자의 전문성을 존중하며 협업하는 개발자가 되고자 합니다. 이를 통해 프론트엔드 개발자로서 팀의 목표 달성에 기여할 수 있다고 확신합니다.
